Introduction

Hoxy is a web debugging proxy API for node. You can use it to debug your production app directly, or as the basis for other development and debugging tools. 

  var hoxy = require('hoxy');

  var proxy = new hoxy.Proxy().listen(8080);

  proxy.intercept('request', function(req, resp){
    console.log('request made to: ' + req.fullUrl());
  });

After running the above code, you can configure your web client to proxy through localhost:8080. All information flowing through the proxy is exposed as JavaScript variables. Use cases include:

  • Inspecting and logging web traffic.
  • Altering the DOM of web pages.
  • Altering JSON requests and responses.
  • Testing JavaScript changes in production.
  • Simulating latency or slow connections.
  • Tampering with HTTP headers.
  • Causing mayhem, chaos, and confusion.

Download & Installation

Hoxy is available as hoxy on npm. You can also check out the source on github. Hoxy depends on node version 0.10.x or higher. 

  npm install hoxy

  var hoxy = require('hoxy');

How it works

Hoxy functions as a normal proxy, standing between the client and the server. You intercept traffic during different phases, such as the request or the response. When you intercept in the request phase, the request object contains data sent to the proxy by the client, but which hasn't yet been sent to the server. During this time, you can inspect or alter any request data. The response object is also available, but unpopulated.

When you intercept in the response phase, the response object contains data sent to the proxy by the server, but which hasn't yet been sent to the client. During this time you can inspect or alter any response data. The request object is still available, but read-only. The phase lifecycle section contains a complete description of the request/response lifecycle, but here's a simplified illustration. 

  -----------------------
  server:       3
  -------------/-\-------
  hoxy:       2   4
  -----------/-----\-----
  client:   1       5
  -----------------------
  1. Client sends request.
  2. Hoxy intercepts request.
  3. Server receives request and sends response.
  4. Hoxy intercepts response.
  5. Client receives response.

API documentaiton

Main module

  var hoxy = require('hoxy');

The object returned from require('hoxy') is the entry point for all hoxy functionality. Here are its properties.

hoxy.Proxy

New proxy constructor. More info below.

hoxy.forever([handler])

Prevents process dying on uncaught errors. In this example, errors are printed to stderr. 

      hoxy.forever();

Alternatively, call callback on uncaught errors. 

      hoxy.forever(function(err){
        console.error(err.stack);
      });

Class Proxy

Represents a proxy server.

new Proxy(options)

Constructs a new proxy with given options. In this example, no options are provided. 

      var hoxy = require('hoxy');
      var proxy = new hoxy.Proxy();

Create a reverse proxy. Now, instead of configuring your client to proxy through localhost:8080, point your client directly at localhost:8080. 

      var hoxy = require('hoxy');
      var proxy = new hoxy.Proxy({
        reverse: 'http://example.com'
      });

Make your proxy use $HTTP_PROXY. 

      var hoxy = require('hoxy');
      var proxy = new hoxy.Proxy({
        upstreamProxy: process.env.HTTP_PROXY
      });

proxy.listen(port)

Starts proxy listening on port. Returns itself. 

      var hoxy = require('hoxy');
      var proxy = new hoxy.Proxy().listen(8080);

proxy.intercept(options, handler)

Entry point for intercepting and operating on requests and responses. This first example intercepts all requests. 

      proxy.intercept('request', function(req, resp){
        console.log('request made to: '+req.url);
      });

This is more verbose, but identical to the above. 

      proxy.intercept({phase:'request'}, function(req, resp){
        console.log('request made to: '+req.url);
      });

Intercept callbacks with arity of three are treated as asynchronous. In this case, call done() to pass control to the next intercept handler. 

      // make everything slow
      proxy.intercept('request', function(req, resp, done){
        setTimeout(done, 1000);
      });

Admit that you screwed up. 

      // sync
      proxy.intercept('request', function(req, resp){
        throw new Error('I screwed up.');
      });

      // async
      proxy.intercept('request', function(req, resp, done){
        done(new Error('I screwed up.'));
      });

Intercept all GETs to example.com. 

      proxy.intercept({
        phase: 'request',
        method: 'GET',
        host: 'example.com'
      }, function(req, resp){
        console.log('request made to: '+req.url);
      });

Intercept text/html responses, exposing the response body as the $ variable. 

      proxy.intercept({
        phase: 'response',
        mimeType: 'text/html',
        as: '$'
      }, function(req, resp){
        resp.$('title')
        .text('all your titles are belong to us');
      });

Intercept UTF-8 responses, exposing the response body as the string variable. 

      proxy.intercept({
        phase: 'response',
        contentType: /utf\-8/i,
        as: 'string'
      }, function(req, resp){
        console.log(resp.string);
      });

Intercept application/json responses, exposing the request body as the json variable. 

      proxy.intercept({
        phase: 'request',
        mimeType: 'application/json',
        as: 'json'
      }, function(req, resp){
        console.log(req.json.email_address);
      });
General options
name type required description
phase string yes Which phase to intercept. See phase lifecycle for more info. Accepted values:
  • request - When proxy has received request.
  • request-sent - When proxy has sent request.
  • response - When proxy has received response.
  • response-sent - When proxy has sent response.
as string no Expose the request or response body (depending on the phase) as data of a certain type. If there's an error parsing the body into this form, the intercept action is skipped and a warning is logged. Hoxy normally streams request and response bodies through. If as is present, hoxy buffers the request or response body into memory. Accepted values:
  • $ - A DOM object similar to jQuery. See cheerio.
  • json - A JS object containing JSON.
  • string - A plain string.
  • buffers - An array of buffers containing the entity body in its raw binary form.
  • params - A JS object containing name/value pairs parsed from the application/x-www-form-urlencoded entity body.
Use filtering in tandem with these. E.g. use $ with mimeType: 'text/html'.
Filtering options (logically ANDed together)
name type required description
protocol string, regexp no Match the request protocol.
method string, regexp no Match the all-uppercase HTTP request method.
host string, regexp no Match the value of the host header.
hostname string, regexp no Match the host, not including :port.
port number, string, regexp no Match the port number.
url string, regexp no Match the request URL. Patterns like /foo/* are allowed. See route-pattern.
fullUrl string, regexp no Match the full request URL including protocol and hostname. Patterns like /foo/* are allowed. See route-pattern.
contentType string, regexp no Match the full content-type header of the request or response (depending on the phase).
mimeType string, regexp no Match just the mime type portion of the content-type header of the request or response (depending on the phase). I.e., if the entire header is "text/html; charset=utf-8", just match the "text/html" part.
filter function no A function that receives req and resp args, returning true or false depending on whether the intercept should be called. Non-async.

proxy.log(levels, [handler])

Deals with various logging events. This first example listens for error, warn, and debug logging events, and prints them to stderr. 

      proxy.log('error warn debug');

Or, print logging events to various writable streams. 

      proxy.log('error warn debug', process.stderr);
      proxy.log('info', process.stdout);

Or, explicitly handle logging events. 

      proxy.log('error warn', function(event){
        console.error(event.level + ': ' + event.message);
        if (event.error) console.error(event.error.stack);
      });
Description of logging events
name description
error When something bad happens that we wish wouldn't have happened.
warn When something iffy happened that we can probably tolerate.
info When something noteworthy happened that normal people care about.
debug When something boring happened that developers care about.

proxy.close()

Stops proxy receiving requests. Finalizes and/or cleans up any resources the proxy uses internally. 

      proxy.close();

Class Request

Represents a request. An instance of this is passed as the first argument to every intercept handler. Altering its values changes what the server sees. If you change the hostname, it changes which server sees the request..

request.protocol

Protocol of the request. Includes a colon but not slashes, like 'http:' Setting this to 'https:' requires providing keys and running the proxy server in encrypted mode.

Note, encrypted mode is not yet implemented. At the moment, only http: is supported.

request.hostname

Destination server hostname, sans port.

request.port

Destination server port.

request.method

All-caps HTTP method used. Lowercase values are converted to uppercase.

request.headers

HTTP request header name/value JS object. These are all-lowercase, e.g. accept-encoding.

request.url

Root-relative request URL, including query string, like /foo/bar?baz=qux

request.json

Request body parsed as JSON. This is only present if you intercept the request as:'json'. Changes made to this object will be seen by the server. 

      proxy.intercept({
        phase: 'request',
        as: 'json',
        method: 'PUT',
        fulUrl: 'http://example.com/users/:id'
      }, function(req, resp){
        // sign everyone up for promo emails
        // whenever they save their preferences
        req.json.prefs.subscriptions = 'all';
      });

request.params

Request body parsed as form-url-encoded params. This will be a key/value POJO. Parameters from the URL querystring will not be included in this object. This object will only present if you intercept the request as:'params'. Changes made to this object will be seen by the server. 

      proxy.intercept({
        phase: 'request',
        method: 'POST',
        mimeType: 'application/x-www-form-urlencoded',
        as: 'params'
      }, function(req, resp){
        console.log(req.params.email);
      });

request.string

Request body string. This is only present if you intercept the request as:'string'. Overwriting this will overwrite the request body sent to the server. 

      proxy.intercept({
        phase: 'request',
        as: 'string',
        url: /\.html$/
      }, function(req, resp){
        // save all pages
        fs.writeFile(req.url, req.string, function(err){
          if (err) console.error('oops!');
        });
      });

request.buffers

Request body binary buffer array. This is only present if you intercept the request as:'buffers'. Changes made to this object will be seen by the server. 

      proxy.intercept({
        phase: 'request',
        as: 'buffers',
        method: 'POST',
        fullUrl: 'http://example.com/images'
      }, function(req, resp){
        // req.buffers contains uploaded image
      });

request.slow(options)

Simulates slowness during request phase. With this method you can set a minimum latency and/or maximum transfer rate. Since these are minimum/maximum, if your native connection is already slower than these values, this method will have no effect. 

      // Simulate upload speed of 10,000 bytes per second
      proxy.intercept('request', function(req, resp){
        req.slow({rate:10000}); // bytes per second
      });
    
      // Simulate a 500-1000ms delay on every request.
      proxy.intercept('request', function(req, resp){
        req.slow({latency:randint(500, 1000)});
      });

request.fullUrl()

Gets the absolute URL of this request. This is mainly a convenience method. 

      // Log every request through the proxy.
      proxy.intercept('request', function(req, resp){
        console.log(req.fullUrl());
      });

Class Response

Represents a response. An instance of this is passed as the second argument to every intercept handler. This object is unpopulated during the request and request-sent phases. Altering its values changes the response to the client.

response.statusCode

HTTP status code being sent to the client.

response.headers

HTTP response header name/value JS object. Header names are all-lowercase, such as 'content-type'.

response.$

Response body parsed as DOM. This object is only present if you intercept the response as:'$'. This is a cheerio object, which provides a jQuery-like API. Changes made to it will be seen by the client. 

      proxy.intercept({
        phase: 'response',
        as: '$',
        fullUrl: 'http://example.com/page.html'
      }, function(req, resp){
        // change the title of the page
        resp.$('title').text('Fake Title!');
      });

response.json

Response body parsed as JSON. This is only present if you intercept the response as:'json'. Changes to this object will be seen by the client. 

      proxy.intercept({
        phase: 'response',
        as: 'json',
        method: 'GET',
        fullUrl: 'http://example.com/users/123'
      }, function(req, resp){
        // arbitrarily manipulate the response json
        resp.json.foo = 'bar';
      });

response.string

Response body string. This is only present if you intercept the response as:'string'. Overwriting this will overwrite the response body sent to the client. 

      proxy.intercept({
        phase: 'response',
        as: 'string',
        fullUrl: 'http://example.com/page.html'
      }, function(req, resp){
        // print page to log
        console.log(resp.string);
      });

response.buffers

Response body binary buffer array. This is only present if you intercept the response as:'buffers'. Changes made to this object will be seen by the client. 

      proxy.intercept({
        phase: 'response',
        as: 'buffers',
        fullUrl: 'http://example.com/image.jpg'
      }, function(req, resp){
        // resp.buffers contains a jpg
      });

response.slow(options)

Simulates a slow response. With this method you can set a minimum latency and/or maximum transfer rate. Since these are minimum/maximum, if your native connection is already slower than these values, this method has no effect. 

      // Simulate download speed of 100,000 bytes per second
      proxy.intercept('response', function(req, resp){
        resp.slow({rate:100000}); // bytes per second
      });
    
      // Simulate a 500-1000ms delay on every response.
      proxy.intercept('response', function(req, resp){
        resp.slow({latency:randint(500, 1000)});
      });

Class Cycle (this)

Represents a single request/response cycle. An instance is this in all intercept handler calls, and the same instance is shared across an entire request/response cycle. It provides a small number of methods not associated specifically to either the request or response.

this.serve(options, callback)

Provisions responses from the local filesystem. Generally, the reason you'd do this is to be able to edit those files locally and test them as if they were live on the remote server. This action populates the response object; see response population for more info. Example. 

      proxy.intercept({
        phase: 'request',
        fullUrl: 'http://example.com/main.js'
      }, function(req, resp, done){
        this.serve('/Users/gr123/test/main.js', done);
      });

Or the more verbose-but-identical... 

      proxy.intercept({
        phase: 'request',
        fullUrl: 'http://example.com/main.js'
      }, function(req, resp, done){
        this.serve({
          path: '/Users/gr123/test/main.js'
        }, function(err){
          done(err);
        });
      });
Options
name type required description
path string no Which file to serve. Defaults to the request URL. Normally this would be used in mutual exclusion with docroot. Strictly speaking, path is always rooted to docroot, which defaults to "/".
docroot string no Which local directory to serve out of. Defaults to filesystem root "/".
strategy string no Describes the relationship between the local docroot and the remote one. Strictly speaking, this controls what happens when the local docroot is missing a requested file. Accepted values:
  • replace - (default) Replaces the remote docroot with the local one. In other words, if a requested file doesn't exist locally, it populates the response with a 404, even if it would have been found remotely.
  • overlay - Overlays the local docroot on top of the remote one. In other words, if a requested file doesn't exist locally, it becomes a no-op, allowing the proxy to "see through" to the one on the remote server.
  • mirror - Automatically mirror the remote docroot locally. In other words, if a requested file doesn't exist locally, it's copied to the local docroot from the remote one, and will be found locally on subsequent requests.

callback is called after the response has been populated. If an error occurred, the error is passed to the callback, otherwise no arguments are passed. Note that a 404 response is not considered an error in this case. There are at least three use cases worth mentioning.

Use case #1: Serve a specific file. 

      proxy.intercept({
        phase: 'request',
        fullUrl: 'http://example.com/js/main.js'
      }, function(req, resp, done){
        this.serve('/Users/bk123/test/main.js', done);
      });

Use case #2: Serve out of a local docroot. 

      proxy.intercept({
        phase: 'request',
        host: 'example.com',
        mimeType: /javascript/i
      }, function(req, resp, done){
        this.serve({
          docroot: '/Users/bk123/test',
          strategy: 'overlay'
        }, done);
      });

Use case #3: Serve out of a local docroot. But automatically copy the remote files to the local docroot the first time they're requested. This is nice if you're one of those "lazy" programmers. 

      proxy.intercept({
        phase: 'request',
        host: 'example.com',
        mimeType: /javascript/i
      }, function(req, resp, done){
        this.serve({
          docroot: '/Users/bk123/test',
          strategy: 'mirror'
        }, done);
      });

this.data(name, [value])

Stores and retrieves data on a cycle instance. This is useful since the same instance is shared across all intercept handlers for a given request/response cycle, allowing you to share related data across disparate scopes. With two params this method behaves as a setter, with one param as a getter. 

      ['request','request-sent','response','response-sent']
      .forEach(function(phase){
        proxy.intercept(phase, function(){
          this.data(phase, Date.now());
        });
      });
      proxy.intercept('response-sent', function(){
        var reqReceived = this.data('request');
        var reqSent = this.data('request-sent');
        var respReceived = this.data('response');
        var respSent = this.data('response-sent');
        // now print some profiling data
      });

Appendix

Phase lifecycle

Full description of phases
# phase description
1 request The proxy has received the request headers, but the request body (if there is one) hasn't started streaming in yet. Or, if you've intercepted a request as:'json' (for example) then the request body will have been fully buffered into memory and be available as request.json (for example). See intercepts for more info.
2 request-sent The proxy has finished sending the entire request, including the request body if present, to the server. Everything is read-only during this phase. The main reason for its existence is to be able to measure the time it takes to upload the request by comparing it with the previous phase.
3 response The proxy has received the response headers, but the response body hasn't started streaming in yet. Or, if you've intercepted a response as:'$' (for example) then the response body will have been fully buffered into memory and be available as response.$ (for example). See intercepts for more info.
4 response-sent The proxy has finished sending the entire response, including the response body, to the client. Everything is read-only during this phase. The main reason for its existence is to be able to measure the time it takes to download the response by comparing it with the previous phase.

Intercept handlers are called in the order of their phase. Intercept handlers of the same phase are called in the order they're declared. request and response objects are either read-only or readable/writable, depending on phase. Attempts to write a read-only thing will fail silently, generating an error log event.

Readability / writability matrix Object
request response
Phase request writable writable
request-sent read-only read-only
response read-only writable
response-sent read-only read-only

Response population

Hoxy normally populates responses by sending the request to the destination server during the normal request/response lifecycle. Alternatively, if you modify any aspect of the response before this happens (i.e. during the request phase) hoxy treats the entire response as populated, and skips the call to the destination server. If you modify any aspect of the response after this happens (i.e. during the response phase) it overwrites just that aspect of the response from the server. 

  proxy.intercept('request', function(req, resp){
    response.string = 'Hello';
    // The response is now populated so the
    // server call is skipped. Status code will
    // default to 200.
  });

  proxy.intercept('response', function(req, resp){
    response.string = 'Hello';
    // The response was already populated,
    // we just overwrite its body. Status
    // code remains the same.
    response.statusCode = 444;
    // Oops, no. We just overwrote the status
    // code. I wonder how the browser will
    // react to this unexpected turn of events.
  });

Change accumulation

Changes to requests and responses are cumulative across all intercepts. Among other things, this affects loading content as a certain type, and filtering.

Change accumulation example. 

  proxy.intercept('request', function(req, resp){
    req.headers['cache-control'] = undefined;
  });

  proxy.intercept('request', function(req, resp){
    console.log(req.headers['cache-control']); // undefined
  });

Change accumulation affects filtering. 

  proxy.intercept('request', function(req, resp){
    req.hostname = 'example.com';
  });

  proxy.intercept({
    phase: 'request',
    hostname: 'other.com'
  }, function(req, resp){
    // never called!
  });

Change accumulation affecting as parameters. 

  proxy.intercept({
    phase: 'response',
    mimeType: 'application/json',
    as: 'json'
  }, function(req, resp){ ... });

  proxy.intercept({
    phase: 'response',
    mimeType: 'application/json'
  }, function(req, resp){
    console.log(typeof resp.json); // 'object'
  });

  proxy.intercept({
    phase: 'response',
    mimeType: 'application/json',
    as: 'string'
  }, function(req, resp){ ... });

  proxy.intercept({
    phase: 'response',
    mimeType: 'application/json'
  }, function(req, resp){
    console.log(typeof resp.json); // 'undefined'
    console.log(typeof resp.string); // 'string'
  });
